How This Program Works

This CAPTCHA application displays five images in a row on an HTML5 Canvas element. Each image is either a cat or a rocket, drawn entirely using standard canvas drawing commands such as arc(), ellipse(), fillRect(), moveTo() and lineTo(). No external images are loaded.

The cat is composed of seven core pieces: a body ellipse, a head circle, two triangular ears, two circular eyes with slit pupils, and a triangular nose. It also features decorative details including pink inner ears, whiskers, a small mouth and a curved tail to make it visually distinctive and recognisable.

The rocket is composed of five core pieces: a body rectangle, a triangular nose cone, two triangular side fins, and a circular porthole window with a glass reflection. Decorative extras include two horizontal body stripes, a rivet on the nose cone, and a three-layered exhaust flame for added visual interest.

Every time the program runs or is reinitialised, three things are randomised: (1) which of the two shapes the user is asked to find, (2) how many of that shape appear (between 1 and 4 out of 5), and (3) which grid positions those shapes occupy. Randomisation of positions is achieved using a Fisher–Yates shuffle algorithm.

Clicking or tapping a cell toggles its selection, indicated by a green semi-transparent overlay, a highlighted cell border, and a white tick badge in the corner. Pressing Verify checks the selection. An edge case is handled: if the user presses Verify without selecting anything, a warning message is shown. A correct answer displays a themed SUCCESS screen with confetti. An incorrect first attempt reshuffles the images for a second try. A second incorrect attempt shows a FAILED screen with a prominent X symbol and locks further input. The Reinitialise button fully resets the program at any time.

Mobile & Desktop Compatibility

The program is designed and tested to work on both mobile and desktop devices. The following techniques are employed:

Touch & click support: The canvas listens for both click events (mouse on desktop) and touchstart events (finger on mobile). The touchstart handler calls preventDefault() to stop the browser from also firing a delayed click event, which would cause the cell selection to toggle twice (back to its original state) on touch devices.

Responsive sizing: The canvas uses a fixed internal coordinate system (650 × 140 pixels) so that all drawing commands produce consistent results. However, the canvas element is styled with max-width: 100%, so it automatically scales down on narrow screens. When a click or touch event occurs, screen coordinates are translated to internal canvas coordinates using getBoundingClientRect() and a computed scale factor. This ensures accurate cell detection regardless of the rendered size.

Accessible touch targets: All interactive buttons have a minimum size of 48 × 48 pixels, following WCAG mobile accessibility guidelines. Each canvas cell measures 130 × 130 pixels in the internal coordinate system, making them comfortably large for finger taps even on small phone screens.

Viewport meta tag: The page includes a <meta name="viewport"> tag set to width=device-width, initial-scale=1.0. This prevents the browser from applying a default desktop-width zoom on mobile devices, ensuring the layout fills the screen naturally on phones and tablets.

Flexible layout: The page uses percentage-based widths with a max-width constraint and flexbox centering, so the content adapts gracefully to any screen size. Buttons use flex-wrap: wrap so they stack vertically if the viewport is too narrow to display them side by side.
